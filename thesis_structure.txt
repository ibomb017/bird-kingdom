[ 240] [sz=  133350] 卷积层：通过可学习卷积核（3×3/5×5 为主）在特征图上滑动做卷积运算，提取局部特征，表达式为：y(i,j)=∑m​∑n​x(i+m,j+n)⋅w(m,n)+b其中，x 为输入特征图，w 为卷积核权重，b 为偏置项，y 为输出特征图，多卷

[ 242] [sz=  133350] 池化层：采用 2×2 最大池化 / 平均池化，对特征图进行下采样，缩小空间尺寸并减少参数计算量，同时增强特征对微小位移的鲁棒性，适配鸟类鸣声时频特征的动态变化。
[ 243] [sz=  133350] 全连接层：将卷积层提取的高级特征映射至分类空间，通过神经元连接综合全局特征信息，完成鸟类物种分类决策。
[ 244] [sz=  133350] Dropout 层：训练过程中随机丢弃 20%~50% 神经元连接，强制网络学习鲁棒特征，有效避免模型过拟合，提升泛化能力。
[ 245] [sz=  133350] 批归一化层（Batch Normalization）：对每批训练数据的中间特征进行均值与方差归一化，加速模型训练收敛速度，同时起到正则化效果，优化模型训练稳定性。

[ 989] [sz=  177800] 6.2 存在的问题与不足
[ 990] [sz=  133350] 尽管系统已完成开发并上线运行，但在开发和测试过程中也发现了一些需要进一步改善的问题：
[ 991] [sz=  133350] （1）鸟类语音识别的鸟种覆盖范围有限。 当前自训练的轻量化模型仅覆盖10种常见鸟类；虽然系统同时集成了支持更广泛鸟种的预训练模型，但两套模型在用户交互层面的无缝切换机制尚未完善。距离覆盖大多数观赏鸟种（约100种以上）还有较大差距。主要受限
[ 992] [sz=  133350] （2）语音识别在复杂噪声环境下的鲁棒性不足。 测试集中的录音大多在相对安静的环境中录制，当录音环境中存在较强的背景噪声（如交通噪声、多只鸟同时鸣叫）时，模型的识别准确率会显著下降。当前的MFCC特征虽然对平稳噪声有一定抗性，但面对非平稳噪声
[ 993] [sz=  133350] （3）离线数据同步机制的冲突处理策略较为简单。 当前采用的"最新修改优先"策略虽然能解决大部分冲突场景，但在多用户共养场景下，如果两个共养者同时离线修改同一只鸟的不同字段，简单的时间戳比较可能导致部分修改被覆盖。更精细的字段级冲突合并还未实
[ 994] [sz=  133350] （4）社交广场缺少智能内容推荐。 当前帖子列表按时间倒序排列，未实现个性化推荐算法。随着用户和内容规模的增长，简单的时间线展示可能无法满足用户发现优质内容的需求。
[ 995] [sz=  133350] （5）系统仅支持iOS平台。 受限于SwiftUI的平台局限性，当前应用仅支持iOS设备，Android用户无法使用。这限制了系统的用户覆盖范围。
[ 996] [sz=  133350] （6）AI问诊功能依赖第三方API。 AI智能问诊模块的回答质量完全取决于所调用的大语言模型API，系统本身不具备针对鸟类健康领域的专业知识训练，可能在某些专业性问题上给出不够准确的建议。
